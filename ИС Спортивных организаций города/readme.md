# Информационная система «Спортивные организации города»

## О проекте

**ИС «Спортивные организации города»** — это реляционная база данных для комплексного учёта и автоматизации деятельности спортивной инфраструктуры города. Система позволяет централизованно хранить информацию о спортивных сооружениях, клубах, спортсменах, тренерах, тренировках, соревнованиях, результатах и награждениях.

Внедрённые триггеры автоматизируют бизнес-логику: автоматическое награждение, повышение спортивных разрядов, проверку соответствия тренера виду спорта, контроль возраста участников и вместимости сооружений. Реализованы 13 аналитических представлений и 13 параметризованных хранимых процедур. Работает полноценная система аудита изменений и архивации удалённых записей.

## 🛠 Технологический стек

- **СУБД:** MySQL
- **Инструмент администрирования:** phpMyAdmin
- **Языки:** SQL (DDL, DML), процедурные расширения (триггеры, хранимые процедуры)
- **Моделирование:** ER-диаграммы (логическая и физическая модели)

## 🗂 Структура базы данных

### Основные таблицы

| Таблица | Назначение |
|---------|-------------|
| `Tip_sooruzheniya` | Типы спортивных сооружений (стадион, бассейн, манеж и т.д.) |
| `Sportivnoe_sooruzhenie` | Спортивные сооружения (название, вместимость, тип покрытия, площадь, адрес) |
| `Vidy_sporta` | Виды спорта |
| `Sportivnye_kluby` | Спортивные клубы |
| `Sportsmeny` | Спортсмены (ФИО, разряд, пол, дата рождения, привязка к клубу) |
| `Trenery` | Тренеры (ФИО, вид спорта, стаж) |
| `Trenirovki` | Тренировки (тренер, спортсмен, вид спорта, дата/время начала, длительность) |
| `Organizatory_sorevnovaniy` | Организаторы соревнований |
| `Sostyazaniya` | Соревнования (название, вид спорта, место проведения, организатор, дата, время, город) |
| `Rezultaty_uchastiya` | Результаты участия (место, баллы, время, лучший результат) |
| `Nagrazhdenie` | Награждения (название награды, дата вручения) |

### Служебные таблицы (аудит и архивация)

- `AuditLog` – журнал всех изменений (INSERT/UPDATE/DELETE) с данными до и после операции, временем и пользователем.
- `Sportsmeny_log` – специализированный лог изменений разрядов спортсменов.
- `Sportsmeny_backup`, `Trenery_backup`, `Sostyazaniya_backup`, `Nagrazhdenie_backup` – архивы удалённых записей с меткой времени.

## ⚙️ Ключевая функциональность

### Триггеры (автоматизация бизнес-процессов)

- **Автоматическое награждение** – при добавлении результата участия создаётся запись в таблице награждений (золото/серебро/бронза/диплом/сертификат).
- **Повышение спортивного разряда** – за 1 место присваивается «Мастер спорта» (МС), за 2-3 места – «Кандидат в мастера спорта» (КМС), если текущий разряд ниже.
- **Проверка соответствия тренера виду спорта** – блокировка создания тренировки, если тренер не специализируется на указанном виде спорта.
- **Проверка возраста спортсмена** – перед добавлением результата проверяется, что возраст участника находится в допустимых пределах (12–60 лет).
- **Проверка вместимости сооружения** – перед добавлением соревнования оценивается планируемое количество участников (упрощённо – 100 человек), и если вместимость меньше – операция отклоняется.
- **Защита от удаления активных тренеров** – запрет на удаление тренера, у которого есть будущие тренировки.
- **Архивация удалённых записей** – перед удалением записи из основных таблиц её копия сохраняется в соответствующую `_backup` таблицу.
- **Логирование изменений** – все операции UPDATE и DELETE фиксируются в `AuditLog` с указанием старых и новых значений.

### Представления (Views) – 13 готовых отчётов

Примеры:

- Спортивные сооружения по типу
- Список спортсменов по виду спорта и разряду
- Спортсмены, занимающиеся более чем одним видом спорта
- Тренеры указанного спортсмена
- Соревнования за произвольный период
- Призёры соревнования
- Активность клубов (количество спортсменов, участников соревнований, победителей)
- Статистика организаторов (количество соревнований, виды спорта, используемые сооружения)
- Спортивные сооружения с датами проведения соревнований

### Хранимые процедуры – 13 параметризованных запросов

Примеры:

- `sp_GetSooruzheniyaPoTipu` – поиск сооружений по типу, минимальной вместимости, типу покрытия, площади.
- `sp_GetSportsmenyPoViduIRazryadu` – спортсмены с фильтрацией по виду спорта и минимальному разряду (с учётом иерархии разрядов).
- `sp_GetSorevnovaniyaZaPeriod` – соревнования за диапазон дат с возможностью фильтрации по организатору.
- `sp_GetSportsmenyBezUchastiya` – спортсмены, не участвовавшие в соревнованиях за указанный период.
- `sp_GetOrganizatoryStatistika` – сводная статистика по организаторам.
- `sp_RestoreSportsmen` – восстановление удалённого спортсмена из архива.
- `sp_GetAuditStatistics` – статистика операций из журнала аудита.
- `sp_ViewRecentLogs` – просмотр последних N записей в логе.

## Система аудита и восстановления данных

- Каждое изменение (`UPDATE`) или удаление (`DELETE`) в ключевых таблицах (`Sportsmeny`, `Trenery`, `Sostyazaniya`, `Rezultaty_uchastiya`) автоматически записывается в таблицу `AuditLog`.
- Для удалённых записей создаются резервные копии в специализированных `_backup` таблицах.
- Хранимые процедуры позволяют:
  - просматривать историю изменений;
  - получать статистику по типам операций и таблицам;
  - восстанавливать удалённых спортсменов из архива.

## Примеры использования

```sql
-- Получить все стадионы с вместимостью более 5000
CALL sp_GetSooruzheniyaPoTipu('Стадион', 5000, NULL, NULL);

-- Показать спортсменов, не участвовавших в соревнованиях в 2024 году
CALL sp_GetSportsmenyBezUchastiya('2024-01-01', '2024-12-31');

-- Просмотреть последние 10 операций в журнале аудита
CALL sp_ViewRecentLogs(10);

-- Восстановить удалённого спортсмена с ID = 5
CALL sp_RestoreSportsmen(5);

-- Получить все соревнования за первый квартал 2025 года
CALL sp_GetSorevnovaniyaZaPeriod('2025-01-01', '2025-03-31', NULL);
```
