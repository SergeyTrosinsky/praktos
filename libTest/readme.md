# Информационная система (Django + PostgreSQL)

## О проекте

Информационная система, объединяющая несколько веб-приложений: библиотечный каталог, новостной блог и систему управления спортивными организациями. Проект реализован на фреймворке Django с использованием СУБД PostgreSQL. Все модули работают в рамках одного сайта, используют общую систему аутентификации и единый стиль оформления.

## Цель

Создать масштабируемую информационную систему с типовыми бизнес-функциями: ведение каталога объектов (книги, новости, спортивные данные), управление пользователями, разграничение доступа и удобная административная панель.

## Задачи

- Разработать модуль «Библиотека» с моделями книг и авторов, возможностью просмотра списка, детальной страницы, добавления новых записей через форму и загрузки обложек.
- Реализовать модуль «Новости» с полным набором CRUD-операций (создание, чтение, обновление, удаление) на основе классовых представлений Django (ListView, DetailView, CreateView, UpdateView, DeleteView).
- Создать модуль «Спортивные организации города» для ведения справочной информации: виды спорта, клубы, объекты, организаторы, соревнования, спортсмены, тренеры, результаты.
- Настроить подключение к PostgreSQL, выполнить миграции, обеспечить целостность данных через внешние ключи.
- Внедрить систему регистрации, входа и выхода пользователей, а также расширенный профиль с возможностью загрузки аватара.
- Обеспечить единый интерфейс с использованием Bootstrap и повторно используемых шаблонов (header, footer).
- Настроить административную панель для управления всеми моделями.

## Технологии

- Python 3
- Django
- PostgreSQL
- psycopg2 (драйвер PostgreSQL)
- Pillow (обработка изображений)
- django-crispy-forms
- Bootstrap 4
- HTML / CSS

## Основные модули

### 1. Библиотека

- Модели: `Author` (автор), `Book` (книга: название, описание, количество страниц, цена, тип обложки, размер, автор, обложка).
- Функционал:
  - Главная страница со списком всех книг (название, автор, обложка-превью).
  - Детальная страница с полной информацией и увеличенным изображением (по клику открывается в новой вкладке).
  - Форма добавления новой книги с выбором автора из выпадающего списка и загрузкой изображения.
  - Загруженные обложки сохраняются в `media/images/`.
- Админка: зарегистрированные модели с настройками фильтрации и поиска.

### 2. Новости (CRUD)

- Модель: `Post` (заголовок, автор (внешний ключ к User), текст).
- Функционал:
  - Список всех новостей на главной странице.
  - Детальная страница отдельной новости.
  - Создание новости (доступно через форму).
  - Редактирование новости.
  - Удаление новости с подтверждением.
- Реализовано на классовых представлениях: `ListView`, `DetailView`, `CreateView`, `UpdateView`, `DeleteView`.
- После создания/редактирования автоматическое перенаправление на детальную страницу (метод `get_absolute_url`).

### 3. Спортивные организации города

- Модели: виды спорта, клубы, спортивные объекты, организаторы, соревнования, спортсмены, тренеры, результаты, связи между соревнованиями и организаторами, тренировки.
- Функционал:
  - Административная панель для управления всеми справочниками.
  - Возможность добавления, редактирования и удаления записей через админку.
  - Поддержка внешних ключей для связывания данных (например, спортсмен → клуб, соревнование → вид спорта).

### 4. Аутентификация и профили пользователей

- Регистрация новых пользователей с валидацией паролей (форма `UserCreationForm` с добавленным полем email).
- Вход и выход из системы.
- Расширенный профиль (модель `Profile` связана с `User` отношением OneToOne):
  - Поле для загрузки аватара (`ImageField` с значением по умолчанию).
  - Отображение имени пользователя, email и аватара на странице профиля.
- Навигационное меню динамически меняется в зависимости от статуса аутентификации (для авторизованных – ссылки «Профиль» и «Выход», для анонимных – «Вход» и «Регистрация»).
- Использование `django-crispy-forms` для стилизации форм.
