\---



\## Описание



Репозиторий содержит выполненные практические задания в рамках учебной практики. В ходе работ реализованы консольные и веб‑приложения, клиент‑серверные системы, интеграция с внешними API, работа с базами данных и создание телеграм‑ботов.



\---



\## Технологии



\- \*\*Python\*\* – основной язык реализации

\- \*\*Django\*\* – веб‑фреймворк (проекты погоды, библиотеки, спортивных организаций)

\- \*\*PostgreSQL\*\* / \*\*SQLite\*\* – реляционные базы данных

\- \*\*FastAPI\*\* – создание собственного веб‑API

\- \*\*pyTelegramBotAPI\*\* – телеграм‑боты

\- \*\*Requests\*\*, \*\*urllib\*\* – HTTP‑клиенты

\- \*\*OMDb API\*\*, \*\*OpenWeatherMap API\*\*, \*\*ЦБ РФ XML API\*\* – внешние сервисы

\- \*\*HTML/CSS (Bootstrap)\*\* – интерфейсы



\---



\## Основные проекты в составе



| Проект | Краткое описание |

|--------|------------------|

| \*\*Кинопоиск через OMDb API\*\* | Консольное приложение с фильтрацией по жанру и типу контента (фильм/сериал). |

| \*\*UDP чат‑бот с шифрованием\*\* | Клиент‑серверный обмен сообщениями с XOR‑шифрованием, команды `/time`, `/usd` (курс ЦБ РФ). |

| \*\*Погодное приложение (Django)\*\* | Веб‑интерфейс для получения прогноза через OpenWeatherMap. |

| \*\*Библиотека (Django + PostgreSQL)\*\* | CRUD для книг, загрузка обложек, админ‑панель, аутентификация пользователей. |

| \*\*Новостной блог (Django)\*\* | Полноценный CRUD на классовых представлениях (ListView, DetailView, CreateView, UpdateView, DeleteView). |

| \*\*Спортивная база данных (SQLite + Telegram‑бот)\*\* | Бот для управления спортсменами, тренерами, соревнованиями. Сложные SQL‑запросы (JOIN, GROUP BY). |

| \*\*FastAPI для спортивной БД\*\* | REST‑эндпоинты (`/sportsmen`, `/clubs`, `/prizers/...`) и интеграция с телеграм‑ботом. |
